# 裁判系统简要设计说明

## 1. 目标

我要做一个基础裁判系统，用来在玩家完成摆盘后，对 `TriggerBox` 中的物品进行自动判定和评分。

这个系统先解决三件事：

1. 检测 `TriggerBox` 内一共有多少个有效物品。
2. 判断这些物品的种类和数量是否符合当前要求。
3. 根据物品摆放的“同心度”计算一个额外分数，最后合成总分。

---

## 2. 要判定什么

以“要求三个松饼、两个水果”为例，系统会检查：

- `TriggerBox` 里是否有且仅有 `3` 个 `Pancake`
- `TriggerBox` 里是否有且仅有 `2` 个 `Fruit`
- 是否混入了不应该出现的其他物品
- 所有物品是否尽量围绕“中心线”摆放

如果数量、种类不对，就扣分或者直接判失败。

---

## 3. 我准备怎么做

### 3.1 结算时主动扫描 TriggerBox 内物品

这里不采用持续监听：

- `OnTriggerEnter`
- `OnTriggerExit`

因为你当前的需求不是“实时更新判定”，而是：

- 玩家摆放结束
- 倒计时结束
- 主动调用一次“开始判定”

所以更适合的方式是：

1. 在 `TriggerBox` 上挂一个判定脚本。
2. 这个脚本提供一个公开函数，例如 `StartJudge()` 或 `JudgeNow()`。
3. 当游戏流程进入“结算时刻”时，由流程管理器调用这个函数。
4. 这个函数一次性扫描当前 `TriggerBox` 范围内所有有效物体。

这样做的优点是：

- 逻辑更简单
- 不需要一直维护进入/离开的状态
- 更符合“倒计时结束后统一打分”的流程

为了避免同一个物体有多个碰撞器导致重复计数，我仍然会按“物体实例”去重，不直接按 `Collider` 数量算。

每个可判定物体都需要有身份标记，例如：

- `Pancake`
- `Fruit`
- `Topping`

这样系统才能知道结算时盒子里分别有多少个什么东西。

---

### 3.2 配方要求检查

我会把需求写成一个简单规则列表，例如：

```text
Pancake: 3
Fruit: 2
```

裁判系统会在 `StartJudge()` 被调用时，把 `TriggerBox` 里当前扫描到的数量和要求进行比较：

- 数量够不够
- 数量有没有超出
- 有没有出现不该出现的类型

这一部分先做成最直接的“按类型计数比对”。

---

### 3.3 同心度评分

同心度的基准这样定：

1. 以当前扫描结果里“最低的那个有效物体”或“最早进入摆盘中心的基准物体”作为基准物体。
2. 取这个物体中心点的位置。
3. 过这个中心点，沿世界坐标 `Y` 轴向上建立一条竖直中心线。
4. 后续所有进入判定区的物体，都计算它们中心点到这条竖直线的水平距离。

理解上就是：

- 如果所有物体都尽量叠在同一条竖线上，分数更高
- 离中心线越远，分数越低

同心度分数会基于“平均偏移量”或“总偏移量”来计算。

简单版本我会先用：

```text
每个物体到中心线的水平距离越小，得分越高
```

---

## 4. 总分怎么组成

先做一个简单总分模型：

```text
总分 = 配方分 + 同心度分
```

其中：

- 配方分：主要看数量和种类对不对
- 同心度分：主要看摆放是否居中、是否整齐

如果后面需要，还可以继续加：

- 高度层次分
- 顺序分
- 时间分

但第一版先不加复杂项。

---

## 5. 我后面要落地成哪些脚本

第一版预计拆成这些脚本：

1. `TriggerBoxJudge`
   - 提供 `StartJudge()` 或 `JudgeNow()`
   - 在结算时扫描盒子内所有有效物体
   - 输出当前物体列表和计数结果

2. `RecipeRequirement`
   - 负责描述当前关卡要求
   - 例如“3 个松饼 + 2 个水果”

3. `JudgeSystem` 或 `ScoreCalculator`
   - 负责把计数结果和配方要求对比
   - 负责计算同心度
   - 输出最终分数

---

## 6. 第一版实现范围

第一版我只会先做基础可用版本：

- 能统计 `TriggerBox` 内物品数量
- 能判断是否满足指定种类和数量要求
- 能基于第一个落地物体的中心线计算同心度
- 能输出一个简单总分

先不处理特别复杂的情况，例如：

- 物体倾斜角度评分
- 多层结构稳定性评分
- 非圆形物体的边缘对齐
- 多个判定区联动

---

## 7. 结果

这套系统做完后，你将能得到一个基础裁判流程：

1. 玩家把物体放进 `TriggerBox`
2. 倒计时结束或流程进入结算阶段
3. 流程管理器调用 `TriggerBoxJudge.StartJudge()`
4. 系统扫描 `TriggerBox` 内所有物体
5. 系统判断是否符合当前要求
6. 系统计算摆放同心度
7. 系统输出总分和判定结果
